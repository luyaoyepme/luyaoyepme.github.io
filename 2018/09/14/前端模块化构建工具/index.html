模块化构建工具

webpack

简介

webpack是模块化管理工具，使用webpack可以对模块进行压缩、预处理、按需打包、按需加载等。

与Grunt/Gulp相比，webpack是一种模块化的解决方案，而后者是一种能够优化前端开发流程的工具，webpack的优点使得webpack在很多场景下可以替代Grunt和Gulp。

webpack的工作方式是：把你的项目做成一个整体，通过一个给定的主文件（如：index.js),webpack将从这个文件开始找到你的项目所有依赖文件，使用loaders处理他们，最后打包成一个（或多个）浏览器可以识别的JavaScript文件。

配置文件

entry

  1. 可以是字符串：这个字符串表示需要打包的模块的路径，如果只有一个要打包的模块，可以使用这种形式

  2. 数组形式，当需要多将多个模块打包成一个模块时，可以是使用这个方式。如果这些模块之间不存在依赖，数组中值的顺序没有要求，如果存在依赖，则要将依赖性最高的放在后面

    例如：[entry:[“./app/one.js”,”./app/two.js”]]

  3. 也可以是键值对形式的对象，当需要分别打包成多个模块时，可以使用这种方式，例如

    当entry是一个键值对形式的对象时，包名就是键名，output的filename不能是一个固定的值，因为每个包的名字不能一样

字符串 或 数组内的文件将合并输出成一个js；对象形式会以key name 分别输出js

output

path:对应一个绝对路径，此路径是你希望一次性打包的目录

__dirname,就是当前webpack.config.js文件所在的绝对路径

publicPath：在编译时不知道最终输出文件的 publicPath 的情况下，publicPath 可以留空，并且在入口起点文件运行时动态设置： __webpack_public_path__。

filename:指定输出文件名

module

test ：一个用以匹配loaders所处理文件的拓展名的正则表达式（必须）

webpack的Loaders功能在modules关键字下进行配置，通过使用不同的loader，webpack有能力调用外部的脚本或工具，实现对不同格式的文件的处理，比如说分析转换scss为css，或者把下一代的JS文件 （es6，es7）转换为兼容现代浏览器的JS文件。

Gulp

简介

Gulp是基于node.js的前端自动化构建工具，前端开发者可以使用它构建自动化工作（前端集成开发环境）等。使用gulp可以简化工作量，让你把重点放在功能的开发上，从而提高开发效率和工作质量，比如，我们可以用gulp可以使网页自动刷新，也可以使用gulp对sass进行预处理、代码检测、图片优化压缩、只需要简单的指令就能全部完成。

Grunt /Gulp的工作方式是：在一个配置文件中，指明对某些文件进行类似编译、组合、压缩等任务的具体步骤，工具之后会自动替你完成这些任务。

gulp的API介绍

gulp.task

执行任务，它又三个参数，其中第二个参数是可选参数。

参数一：要执行的任务名称

参数二：在执行该任务前，需要先执行的任务列表，以数组的形式。

参数三：回调函数

语法：

gulp.src

根据路径找打对应的文件。可以被pipe到其他插件中

语法：

gulp.dest

输出到对应的路径，可以被pipe进来。

语法：

gulp.watch

监听路径文件的变化，执行相应的任务。
